## ¿Qué salió distinto a lo planeado?

La planificación inicial contemplaba crear un dungeon crawler 2D completamente desde cero, incluyendo el sistema de enemigos, trampas, cofres, llaves, puertas, interfaz de usuario y varios niveles conectados entre sí. Sin embargo, durante el desarrollo surgieron diversos obstáculos: algunas mecánicas de IA enemiga resultaron más complejas de lo previsto, aparecieron errores NullReference por sistemas que no estaban correctamente sincronizados, se presentaron fallas al generar la build para WebGL y también hubo problemas con la estructura ZIP necesaria para publicar el juego en itch.io. Por ello, el proyecto tuvo que reorganizarse y apoyarse en assets y sistemas más estables, lo que permitió obtener un resultado final más sólido, visualmente agradable y listo para ser publicado. Además, la inteligencia artificial también colaboró en la creación de iconos, sprites y en la corrección de errores.

## ¿Qué características se eliminaron para cumplir la fecha de entrega?


Para cumplir con la fecha de entrega, fue necesario eliminar varias características que originalmente estaban planificadas. Se descartó el sistema propio de rotación de trampas, un panel de interfaz con animaciones avanzadas, una escena de transición en la puerta final del dungeon, varios niveles que aún estaban en fase de prototipo y un menú principal más complejo con selección de niveles. Al dejar de lado estos elementos, el equipo pudo enfocarse en terminar un nivel de dungeon completamente funcional, bien diseñado y acorde con el tiempo disponible.
